Fundamentos de HTML

HTML (HyperText Markup Language) es el lenguaje de marcado con el que se estructura el contenido de la web. En esta sección cubrimos etiquetas básicas, estructura semántica, formularios y SEO básico en HTML.

Estructura básica y etiquetas semánticas

Un documento HTML5 comienza con <!DOCTYPE html> y se organiza con <html>, <head> y <body>. Dentro del <body> usamos etiquetas semánticas como <header>, <nav>, <main>, <section>, <article>, <aside> y <footer> para dar significado al contenido.

Por ejemplo, a continuación vemos una estructura básica semántica:

<header>
  <h1>Título</h1>
  <nav>...</nav>
</header>
<main>
  <section>
    <h2>Sección</h2>
    <p>Contenido...</p>
  </section>
  <aside>Contenido relacionado...</aside>
</main>
<footer>Pie de página</footer>

Esta estructura mejora la accesibilidad y el SEO, ya que los navegadores y motores de búsqueda entienden la jerarquía del contenido.

Formularios y validación en HTML5

HTML5 introdujo atributos de validación como required, pattern, min, max, etc., que permiten validar datos de formularios en el navegador sin usar JavaScript.

Contacto

En el formulario de ejemplo, todos los campos son obligatorios (atributo required) y el campo de email debe tener formato válido gracias a type="email". Al intentar enviar sin rellenar los campos correctamente, el navegador mostrará errores de validación.

SEO básico en HTML

Para SEO, es importante definir un <title> adecuado, usar meta descripción (<meta name="description">) y estructurar el contenido con encabezados jerárquicos (<h1>...<h2>, etc.). También usar texto alternativo en imágenes (alt) y enlaces con texto descriptivo.

Ejemplo de meta descripción en esta página (ver código fuente en el <head>):

<meta name="description" content="Guía integral sobre los fundamentos de HTML y CSS...">

Con estas prácticas, los motores de búsqueda pueden entender mejor de qué trata la página y mostrarla pertinentemente en resultados.

Fundamentos de CSS

CSS (Cascading Style Sheets) es el lenguaje de estilos que se utiliza para describir la presentación visual de un documento HTML. Aquí veremos selectores, el modelo de caja, diseño responsive con media queries, Flexbox, Grid y pseudo-clases/animaciones.

Selectores en CSS

Un selector CSS permite apuntar a elementos HTML específicos para aplicarles estilos. Ejemplos:

  • elemento: Selecciona todas las etiquetas de ese tipo (ej: p selecciona todos los párrafos).
  • .clase: Selecciona todos los elementos con esa clase (ej: .destacado).
  • #id: Selecciona el elemento con ese id (ej: #menu).
  • padre hijo: Selección por jerarquía (ej: main p párrafos dentro de <main>).
  • Pseudo-clases: ej: a:hover (cuando el cursor está sobre un enlace).

La especificidad y el orden determinan qué estilos prevalecen en caso de conflicto (última regla aplicada y/o selector más específico gana).

Modelo de caja (Box Model)

Todo elemento HTML se representa como una caja con contenido, padding (relleno), borde y margen. Por defecto, width y height aplican al contenido; el padding y border se suman aumentando el tamaño total. Podemos usar box-sizing: border-box; para incluir padding y border en el tamaño definido.

Ejemplo: si a un <div> le damos width: 200px; padding: 10px; border: 2px solid;, el ancho total será 200 + 2*10 + 2*2 = 224px.

Comprender el modelo de caja es crucial para ajustar correctamente los espacios y alineaciones en la página.

Diseño responsive con media queries

Las media queries permiten aplicar estilos dependiendo de las características del dispositivo o ventana. Por ejemplo, podemos cambiar el layout si la pantalla es menor a cierta anchura:

@media (max-width: 600px) {
  /* estilos para móviles */
  body {
    font-size: 16px;
  }
}

Así adaptamos la interfaz a móvil, tablet o desktop. En esta página, utilizamos media queries para que la barra de navegación y las secciones se ajusten en pantallas pequeñas (por ejemplo, el menú se verticaliza en móvil).

Layouts con Flexbox y Grid

Flexbox nos facilita diseñar layouts en una dimensión (fila o columna). Por ejemplo, una barra de navegación horizontal o centrar un elemento verticalmente dentro de su contenedor. Simplemente aplicando display: flex; al contenedor y usando propiedades como justify-content (alineación horizontal) y align-items (alineación vertical) podemos lograr distribuciones comunes.

Grid es aún más poderoso para diseños de dos dimensiones (filas y columnas). Definimos un contenedor con display: grid; y establecemos columnas y filas con grid-template-columns y grid-template-rows. Podemos colocar los ítems en posiciones específicas de la cuadrícula. Es muy útil para maquetados de páginas completos o galerías.

En esta guía usamos Flexbox para la sección de navegación (distribuyendo los enlaces) y Grid para los ejemplos profesionales (tres columnas en desktop, que pasan a una columna en móvil).

Pseudo-clases y animaciones

Las pseudo-clases como :hover, :focus, :first-child, etc., nos permiten aplicar estilos en estados particulares. Ejemplo: a:hover { color: orange; } cambia el color de un enlace al pasar el cursor.

Para animaciones, CSS ofrece transiciones y animaciones con keyframes. Una transición (transition) permite que un cambio de estilo (por ejemplo, de color o de tamaño) ocurra suavemente en un intervalo de tiempo. Las animaciones con @keyframes permiten definir cambios de estilo en porcentajes de tiempo y se pueden ejecutar en bucle o una vez.

En esta página se han aplicado transiciones simples. Por ejemplo, los botones tienen transition: background-color 0.3s; para que al hacer hover el cambio de color sea gradual. Esto mejora la experiencia de usuario con pequeñas pistas visuales interactivas.

Integración de IA en el aprendizaje

Las herramientas de Inteligencia Artificial pueden apoyar el aprendizaje de HTML y CSS. Por ejemplo, asistentes como ChatGPT pueden responder dudas ("¿Cómo centro un div?") o generar código de ejemplo a partir de indicaciones en lenguaje natural.

Existen generadores de código que, dada una descripción, producen código HTML/CSS. Estas herramientas permiten experimentar y obtener soluciones que luego el estudiante puede analizar. Es importante usarlas como complemento, comparando sus sugerencias con la documentación oficial y probando por uno mismo.

También hay asistentes integrados en editores de código (por ejemplo, GitHub Copilot) que sugieren código mientras escribes. Por ejemplo, al empezar a escribir <form> podrían autocompletar los campos comunes de un formulario.

Recomendación: usar la IA para reforzar el aprendizaje, no para evitarlo. Se puede pedir que explique por qué cierto CSS no funciona, o pedirle consejos de accesibilidad. Luego, verificar siempre en la práctica real lo obtenido. Las IAs son excelentes para obtener pistas rápidas, pero no siempre su output es perfecto u óptimo, por lo que el criterio del desarrollador sigue siendo clave.

Ejercicio propuesto: solicitar a ChatGPT que genere un pequeño código (como una tarjeta de presentación con HTML y CSS), y luego tratar de entender cada línea de lo que generó, modificándolo para personalizarlo. Esto combina lo mejor de ambos mundos: la rapidez de la IA y la comprensión activa del estudiante.

Ejemplos de uso profesional

A continuación, ejemplos resumidos de cómo se aplican HTML/CSS en contextos típicos de los ciclos formativos:

ASIR

Administración de Sistemas: crear la interfaz de una intranet de empresa o una página de monitorización de servidores. Un admin de sistemas puede usar HTML para estructurar datos (tablas de estado, formularios de configuración) y CSS para resaltar alertas (como servidores caídos en rojo). También podría personalizar la apariencia de un gestor de tickets interno usando estos conocimientos.

DAW

Desarrollo Web: diseño del frontend de un sitio o aplicación web. Un desarrollador DAW maquetará con HTML5 semántico un portal (cabecera, sección de noticias, sidebar de widgets, pie). Aplicará CSS (quizás con un framework) para lograr un estilo profesional y responsive. Por ejemplo, en un proyecto e-commerce, crear la página de catálogo de productos con grid CSS para las tarjetas de producto, y formularios de compra accesibles.

DAM

Multiplataforma: creación de interfaces híbridas. Por ejemplo, usando Electron para una app de escritorio, se diseña la ventana principal con HTML/CSS. O en desarrollo móvil híbrido, se construyen pantallas de la app con tecnologías web. El conocimiento de responsive design ayuda a que la misma base HTML/CSS se adapte a distintas resoluciones de dispositivos. Un caso práctico: una app móvil de agenda donde la vista de contactos es una lista

    estilizada con CSS para parecer elementos nativos.

Ejercicios prácticos y retos

  1. Estructura semántica: Maquetar con HTML5 una página básica (cabecera, contenido con secciones y aside, pie de página) para una noticia o tema de tu elección. Asegúrate de usar etiquetas semánticas adecuadas y validar el HTML.
  2. Formulario de contacto: Crear un formulario con campos Nombre, Email, Teléfono y Mensaje. Implementar validación HTML5 (required, type, pattern) y probar los mensajes de error nativos del navegador.
  3. Diseño responsive: Tomar la página del ejercicio 1 o 2 y añadir CSS para que sea responsive. Ej: en móvil, convertir un layout de dos columnas en uno de una columna. Utiliza media queries para lograrlo.
  4. Flexbox: Con CSS Flexbox, centrar un elemento en pantalla (horizontal y verticalmente) y crear una barra de navegación horizontal que distribuya equitativamente 5 enlaces. Luego hacer que en pantalla pequeña los enlaces se apilen verticalmente.
  5. Grid: Construir una galería de 6 imágenes usando CSS Grid en 3 columnas x 2 filas en desktop, que pase a 2 columnas x 3 filas en dispositivos más pequeños. Añadir algo de espacio (gap) entre las imágenes.
  6. Bootstrap: Rehacer el formulario del ejercicio 2 utilizando exclusivamente clases de Bootstrap para el diseño (usa su grid para organizar campos en filas/columnas y clases de formulario para inputs). Compara la velocidad de desarrollo con respecto a escribir CSS manual.
  7. Tailwind: Si te es posible, intenta estilizar un componente (por ejemplo, una tarjeta con título, imagen y texto) usando clases de utilidad de Tailwind en lugar de CSS convencional. Analiza la legibilidad del HTML resultante.
  8. ChatGPT al rescate: Pregúntale a una IA como ChatGPT cómo resolver un problema de CSS que te haya costado (por ejemplo, "¿cómo puedo hacer un div translúcido encima de una imagen de fondo?"). Aplica su solución y verifica si funciona; si no, depúrala tú mismo. Reflexiona sobre lo que hizo igual o diferente a lo que intentaste inicialmente.

Estos ejercicios cubren teoría y práctica. Se recomienda realizarlos y luego compartir el código en un repositorio (por ejemplo, en GitHub) para obtener retroalimentación de compañeros o tutores.

Recursos y referencias adicionales

  • MDN Web Docs - HTML (en español) – Documentación completa de HTML5, con tutoriales y referencias de todas las etiquetas.
  • MDN Web Docs - CSS (en español) – Guía y referencia de CSS, incluyendo secciones dedicadas a Flexbox, Grid, etc.
  • LenguajeHTML.com – Tutoriales en español (por Manz) sobre HTML5, CSS3 y más, muy didácticos y actualizados.
  • Documentación de Bootstrap 5 (oficial) – Para aprender a usar el framework Bootstrap y ver ejemplos de código.
  • Documentación de Tailwind CSS (oficial) – Guía de utilidades CSS de Tailwind y playground en línea para probar clases rápidamente.
  • freeCodeCamp – Plataforma gratuita (disponible en español) con un currículo interactivo para aprender HTML, CSS y desarrollo web en general, mediante ejercicios prácticos.
  • web.dev - Aprender CSS – Curso gratuito de Google (en inglés, con opción en español) sobre fundamentos de CSS, muy actualizado y con buenas prácticas de rendimiento y accesibilidad.
  • W3Schools – Referencia rápida de HTML, CSS, JavaScript, etc. (en inglés sencillo). Útil para consultar ejemplos rápidos de sintaxis.
  • W3C Markup Validator – Herramienta para validar código HTML/XHTML. Ideal para comprobar que tu HTML no tiene errores de sintaxis.
  • CSS-Tricks: Guía de Flexbox – (inglés) Un cheat-sheet visual para entender todas las propiedades de Flexbox.
  • CSS-Tricks: Guía Completa de Grid – (inglés) Similar al anterior pero para CSS Grid, con ilustraciones de cada concepto.
  • Wave (WebAIM) – Herramienta online para evaluar la accesibilidad de tu página web, muy útil tras terminar tu estructura HTML/CSS.

Estos recursos ayudan a ampliar conocimientos. Se recomienda especialmente MDN en español para profundizar en cada tema visto en la guía, y freeCodeCamp para practicar construyendo proyectos completos. ¡Feliz aprendizaje web!
